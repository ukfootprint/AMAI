# operations/ — Schema Reference

All operations files are Markdown. No JSONL or YAML data files in this module. No field-level validation required — validation is by convention and regular review.

---

## workflows.md

Free-form Markdown. Recommended structure: one section per recurring workflow, each with a trigger, steps, and expected output. No fixed schema.

---

## tools.md

Free-form Markdown. Recommended structure: one section per tool category (communication, writing, project management, etc.) with the current tool and brief rationale. No fixed schema.

---

## rituals.md

Free-form Markdown. Structured by cadence (daily, weekly, monthly, quarterly). Each ritual should specify: when, duration, and what it produces. Non-negotiable rituals should be marked explicitly.

---

## MAINTENANCE_BUDGET.md

Reference document — no data fields. Updated when maintenance tiers change.
