# knowledge/domains/ai_infrastructure/

This directory holds knowledge specific to the AI & Personal Infrastructure domain.

## What Goes Here

Domain-specific files that supplement (not replace) the general `knowledge/` files:

| File | Purpose |
|------|---------|
| `frameworks.md` | AI/infrastructure mental models — how to evaluate AI tools, context architecture, automation tradeoffs |
| `landscape.md` | Current AI landscape — capability trends, key players, emerging patterns in personal AI infrastructure |
| `terminology.md` | Domain vocabulary — context window, RAG, prompt engineering, agentic systems, MCP, etc. |
| `resources.md` | Key references — papers, tools, practitioners worth following |

## How These Files Are Used

When a task relates to AI tools, context systems, AMAI itself, or automation — load these files **alongside** the general knowledge files:

- `knowledge/frameworks.md` — general mental models (always load)
- `knowledge/domains/ai_infrastructure/frameworks.md` — AI-specific models (load when domain is ai_infrastructure)
- `knowledge/domains/ai_infrastructure/landscape.md` — current AI landscape context (load for strategy/analysis tasks)

The general files apply to all domains. Domain-specific files add depth for domain-relevant tasks.

## Populating This Directory

These files start as placeholders. Populate them through:
- Organic updates as you learn — this domain moves fast
- After significant AMAI build sessions that reveal new architecture patterns
- The onboarding skill Stage 2 knowledge section

Set `last_updated` in `domain_index.yaml` once the files contain real data.
