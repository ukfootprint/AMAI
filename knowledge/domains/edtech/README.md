# knowledge/domains/edtech/

This directory holds knowledge specific to the Education Technology domain.

## What Goes Here

Domain-specific files that supplement (not replace) the general `knowledge/` files:

| File | Purpose |
|------|---------|
| `frameworks.md` | EdTech-specific mental models — market dynamics, procurement cycles, adoption patterns, policy levers |
| `landscape.md` | Current EdTech market state — players, trends, regulatory environment, UK education policy context |
| `terminology.md` | Domain vocabulary — MIS, MAT, academy trust, Pupil Premium, EEF, Ofsted, SIMS, etc. |
| `resources.md` | Key references — reports, research, publications, people worth tracking |

## How These Files Are Used

When a task relates to EdTech — client work, strategy, competitive analysis, product thinking — load these files **alongside** the general knowledge files:

- `knowledge/frameworks.md` — general mental models (always load)
- `knowledge/domains/edtech/frameworks.md` — EdTech-specific models (load when domain is edtech)
- `knowledge/domains/edtech/landscape.md` — current market context (load for strategy/analysis tasks)

The general files apply to all domains. Domain-specific files add depth for domain-relevant tasks.

## Populating This Directory

These files start as placeholders. Populate them through:
- The onboarding skill Stage 2 knowledge section
- Organic updates after relevant work or conversations
- The `/amai:knowledge-update` command (if configured) after a domain-relevant session

Set `last_updated` in `domain_index.yaml` once the files contain real data.
